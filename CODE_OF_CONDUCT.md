# Code of conduct

## Our commitment

We aim for a **harassment-free, professional** experience for everyone contributing to Autonomous EHS—whether as an employee, contractor, open-source contributor, or partner.

## Expected behavior

- Use welcoming and inclusive language; assume good intent; disagree on substance, not personality.
- Respect confidentiality of **non-public customer/regulated data**; do not paste production PII into issues or chats.
- Follow the security reporting path in [`SECURITY.md`](SECURITY.md) for vulnerabilities.

## Unacceptable behavior

Harassment, discrimination, threats, sustained disruption, or publishing private information without permission will not be tolerated.

## Reporting

For this repository, open a **private** channel with the maintainers (org security or HR per your employer policy). If you are an external contributor without a contact, use GitHub **private vulnerability reporting** only for security defects; for conduct concerns, email the address your org lists in the repo **About** section or contractual support route.

## Enforcement

Maintainers may remove, edit, or reject contributions and temporarily or permanently block anyone whose behavior violates this code. Interpretation and enforcement follow applicable law and organizational HR policies.

## Attribution

Inspired by common open-source norms (e.g. Contributor Covenant). Adapt wording with counsel for your legal entity if you fork this policy.
